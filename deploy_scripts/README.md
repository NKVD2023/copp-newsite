# Скрипты развертывания и обновления (COPP)

В этой папке собраны все скрипты, необходимые для управления сайтом на сервере. 
Для удобства все они запускаются из этой папки, но сами скрипты автоматически переходят в корень проекта, поэтому их не нужно переносить обратно.

## Описание файлов

### 1. `push_code.bat` (Основной рабочий скрипт)
Самый главный и удобный скрипт для ежедневной работы.
**Зачем нужен:** Автоматизирует весь цикл обновления сайта через Git.
**Что делает:**
- Собирает все изменения в вашем локальном коде.
- Отправляет их на GitHub (сразу в рабочую ветку `main`).
- Автоматически подключается к серверу по SSH и дает команду обновить код.
**Как использовать:** Просто дважды кликните по файлу в Windows.

### 2. `deploy_direct.bat` (Экстренное обновление)
Альтернативный скрипт развертывания "в обход" Git.
**Зачем нужен:** Если Git сломался, сервер не имеет доступа к интернету, или вам нужно срочно залить локальные изменения прямо на сервер без истории коммитов.
**Что делает:** Упаковывает весь код проекта в архив, отправляет его по SSH и распаковывает поверх существующих файлов на сервере.
**Как использовать:** Дважды кликните в Windows.

### 3. `update.sh` (Серверный скрипт обновления)
Этот скрипт находится и работает **на самом сервере** (вызывается автоматически скриптом `push_code.bat`).
**Зачем нужен:** Управляет процессом обновления сайта (Gunicorn, зависимости, Nginx).
**Что делает:**
- Скачивает последние обновления с GitHub.
- Восстанавливает папку `.git`, если она была случайно удалена.
- Устанавливает новые зависимости из `requirements.txt`.
- Перезапускает сервисы.
**Как использовать вручную (при необходимости на сервере):**
`sudo bash /var/www/copp-newsite/deploy_scripts/update.sh`

### 4. `deploy.sh` (Скрипт первичной установки)
Этот скрипт используется только один раз при переезде на **совершенно новый сервер**.
**Зачем нужен:** Превращает "пустой" сервер на Ubuntu в готовый веб-сервер для проекта COPP.
**Что делает:** Устанавливает Python, Nginx, настраивает системные службы, SSL-сертификаты, права доступа и загружает код с GitHub.
**Как использовать:** Скопировать на новый сервер и запустить `sudo bash deploy.sh`.

---

**Важное замечание про базу данных и файлы окружения:**
Файлы `coppdb.sqlite` и `.env` добавлены в `.gitignore`. При использовании `push_code.bat` они не отправляются в GitHub и не затираются на сервере. Если вы хотите обновить саму базу данных на сервере, воспользуйтесь `deploy_direct.bat`.
